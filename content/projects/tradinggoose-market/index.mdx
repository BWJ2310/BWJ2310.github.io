<ProjectSection
  eyebrow="System Boundary"
  title="Market records what exists; clients render provider symbols"
  summary="The service avoids locking provider-specific string formatting into the core reference-data platform."
>
  <ProjectSectionCopy>
    A listing keeps its canonical context: base ticker, quote currency, asset class, market, country, city, and related exchange metadata. Each provider can then define rules that score matching contexts and render the final request symbol with templates.
  </ProjectSectionCopy>
  <ProjectSectionMedia>
    <ProcessTimeline steps='[{"title":"Resolve listing context","summary":"Fetch canonical listing metadata from TradingGoose-Market."},{"title":"Match provider rules","summary":"Filter active rules by market, country, city, asset class, currency, or regex."},{"title":"Score precedence","summary":"Rank rules so the most specific provider mapping wins."},{"title":"Render symbol","summary":"Interpolate the winning template into Yahoo, Finnhub, Alpaca, or another provider format."}]' />
  </ProjectSectionMedia>
</ProjectSection>
